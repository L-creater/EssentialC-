虚指针和虚表

如下图所示：
仔细看图

  • A 有A::data1和A::data2
  • B 有A::data1、A::data2和B::data3
  • C 有A::data1、B::data2、B::data3、C::m_data1、C::m_data4
  1. 虚指针指向虚表，就算类有10000个虚函数，虚指针只有一个。
  2. 在计算A的大小时，会发现其大小是所有数据加起来后再加上一个指针的大小。
  3. 继承会把数据继承下来，还会把函数继承下来 —继承的是调用权不是内存大小
  4. 没有虚函数，不会有虚指针。
  5. 虚指针大小为8。
  6. 多态，虚函数，动态绑定都是一种走的是虚指针虚表

class Base {
   public:
   Base() : x(0){};
   virtual void print() { printf("print my value %d\n", x); }

   private:
   int x;
   // int y;
   // double value;
};

class Driver : public Base {
   public:
   virtual void print() { printf("print my value %d\n", xl); }

   private:
   std::string name;
   int xl = 3;
};

class Driver2 : public Base {
   public:
   virtual void print() {
      printf("print my value %d, name %s \n", xl, m_name.c_str());
   }

   private:
   std::string m_name = "asd";
   int xl = 4;
};

int main() {

   Base *base2 = new Driver();
   std::vector<Base *> vec;
   
     Base *base3 = new Driver2();

   vec.push_back(base2);
   vec.push_back(base3);
   // 此处就是多态机制
   vec[0]->print();
   vec[1]->print();
   return 0;
}
// cout：
// print my value 3
// print my value 4, name asd 
//具体调用那个函数--看p指向谁 --这个机制就是多态
//静态绑定一定指向某个地址，动态绑定要看p指向的谁--a b c ？

Dynamic Binding

调用动态绑定的三个条件

  • 通过指针
  • 向上转型–A *p = new B
  • 调用虚函数

B b;
A a = (A)b;
a.vfunc1(); //静态绑定

A *pa = new B;
pa->vfunc1();
